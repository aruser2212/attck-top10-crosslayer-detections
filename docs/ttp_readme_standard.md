# TTP README Standard

Каждая техника описывается по одному компактному шаблону:

1. **Коротко** — что делает техника и почему она важна.
2. **Attack flow** — 3–4 шага цепочки атаки.
3. **Телеметрия** — минимальные источники логов и события.
4. **Detection strategy** — логика сигналов, не привязанная к SIEM.
5. **Реализовано в проекте** — DetCard, Sigma, Hunt, IDPS/YARA, tests.
6. **False positives / tuning** — шум и способы его срезать.
7. **Response checklist** — что делать аналитику после срабатывания.
8. **References** — MITRE, вендорская документация и первичные источники.

Цель — дать полноценный разбор без превращения README в кирпич. Если техника требует длинного анализа, он выносится в `docs/` внутри папки техники.
